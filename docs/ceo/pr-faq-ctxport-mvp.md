## 1. Press Release（新闻稿）

### CtxPort 发布：AI 时代的剪贴板——一键复制 AI 会话为可迁移的 Context Bundle

**2026 年 Q2，线上发布**

CtxPort 今日发布其首款产品——一个 Chrome 浏览器扩展，让用户能从 ChatGPT 和 Claude 的网页端一键复制 AI 会话为结构化的 Markdown Context Bundle，方便喂给任何其他 AI 工具。CtxPort 的所有数据处理完全在用户本地完成，绝不上传任何内容到外部服务器，核心逻辑开源可审计。

**这个产品是为谁做的？**

每天在 ChatGPT、Claude、Cursor、Claude Code 等多个 AI 工具之间切换的开发者和 AI 重度用户。调研数据显示，84% 的开发者使用 AI 工具，60% 以上同时使用多个 AI 平台，每次工具切换平均浪费 15-30 分钟重建上下文，年化浪费超过 200 小时。

**为什么需要 CtxPort？**

目前 AI 平台将上下文视为竞争壁垒，刻意不提供互操作性。用户被迫在平台间手动 copy-paste——对话结构丢失、代码格式损坏、敏感信息暴露风险高。2025 年 12 月的浏览器扩展数据泄露事件（90 万用户数据被窃取）更让用户对第三方工具信任降至冰点。CtxPort 用本地处理和开源透明直接回应这一信任危机。

**核心功能**

- **一键复制当前会话**：在 ChatGPT/Claude 页面点击一个按钮，整个会话立刻变成干净的 Markdown Context Bundle，复制到剪贴板
- **左侧列表不打开就能复制**：在会话列表中，每条会话旁都有复制按钮，不需要打开就能复制——将 4 步操作降到 1 步。这是目前所有竞品都没有的功能
- **批量多选复制**：勾选 10 条会话，一键合并为一个 Context Bundle
- **复制格式选项**：原文 / 仅用户消息 / 仅代码块 / 精简版，满足不同场景需求
- **零上传、本地处理、开源核心**：数据永远不离开用户电脑

CtxPort 创始人表示：

> "我们不是在做又一个 AI Chat Exporter。我们在建设 AI 时代的基础设施——让上下文像水一样在工具间自由流动。用户不应该为'把想法从一个 AI 搬到另一个 AI'付出任何代价。而且，我们坚持一个不可妥协的原则：你的 AI 对话永远只属于你自己。"

来自早期用户、全栈开发者李明（化名）的反馈：

> "我每天在 ChatGPT 里讨论架构方案，然后切到 Claude Code 实现，每次切换都要花 15 分钟重新解释项目背景。装了 CtxPort 之后，我在 ChatGPT 左侧列表勾选相关的 3 个会话，一键复制，粘贴到 Claude Code——AI 立刻理解了之前的完整上下文。最让我安心的是，我检查了源码，确认所有处理都在本地完成。这才是 2026 年该有的体验。"

## 2. Internal FAQ（内部 FAQ）

### Q1：这和 ChatGPT to Markdown、AI Chat Exporter 等竞品有什么不同？

**本质区别在于产品定位的不同。**

现有工具做的是"导出"——把会话变成一个文件存到本地。CtxPort 做的是"上下文打包"——把会话变成一个结构化的、可以直接喂给其他 AI 的 Context Bundle。

具体差异：

| 维度 | 现有竞品 | CtxPort |
|------|---------|---------|
| **核心动作** | 导出为文件（PDF/MD/TXT） | 复制到剪贴板，直接粘贴使用 |
| **列表级操作** | 必须打开会话才能导出 | 左侧列表一键复制，不用打开 |
| **批量能力** | 无批量或极有限 | 多选 + 合并为单一 Bundle |
| **格式选项** | 单一格式 | 原文/仅用户/仅代码/精简版 |
| **数据安全** | 部分需上传到服务器 | 完全本地处理，零上传 |
| **开源** | 大多闭源 | 核心逻辑开源可审计 |

**最关键的差异是左侧列表复制按钮和批量多选。** 这把"复制 5 个会话"的操作从"打开 5 次、复制 5 次、切换 5 次"变成"勾选 5 个、点击 1 次"，是一个数量级的效率提升。

### Q2：为什么只做 ChatGPT + Claude，不做更多平台？

**Speed over scope。先做深，再做广。**

三个原因：

1. **用户覆盖率最高**：Stack Overflow 2025 Survey 显示 82% 开发者使用 ChatGPT，41% 使用 Claude。两个平台覆盖了 90%+ 的目标用户，边际收益最大。

2. **技术复杂度可控**：每个平台的 DOM 结构、API 模式、反爬策略都不同。2 个平台可以做到极致打磨，5 个平台就会陷入"适配器地狱"——每次平台更新 DOM，扩展就崩。

3. **验证核心假设**：MVP 阶段要验证的核心假设是"用户是否真的需要结构化的上下文打包"，而不是"我们能接多少平台"。如果在 ChatGPT + Claude 上证明了需求，扩展到 Gemini、Grok 只是增加一个 Adapter 的工作量。

**后续节奏**：MVP 验证后，第二批支持 Gemini + DeepSeek；同时开放 Adapter SDK 让社区贡献更多平台支持。

### Q3：数据安全如何保证？

**这是我们的第一性原则，不是功能，是产品的宪法。**

三道防线：

1. **架构级保证——零上传**
   - 扩展不请求任何网络权限（no `http://` or `https://` permissions for external servers）
   - 所有 HTML 解析、Markdown 转换、格式处理都在浏览器本地的 Service Worker / Content Script 中完成
   - 用户可以通过 Chrome DevTools Network 面板验证：CtxPort 不发出任何外部网络请求

2. **代码级保证——开源可审计**
   - 核心导出逻辑 100% 开源，MIT License
   - 任何用户、安全研究者、企业 IT 团队都可以审计代码
   - 每个版本发布附带构建可复现性验证（reproducible build）

3. **权限级保证——最小权限**
   - 扩展仅申请对 `chat.openai.com` 和 `claude.ai` 两个域名的 Content Script 注入权限
   - 不申请 `tabs`、`history`、`cookies`、`webRequest` 等高风险权限
   - Chrome Web Store 审核流程保证权限声明的一致性

**这个策略直接回应了 2025 年 12 月的浏览器扩展安全事件。** 那次事件中，恶意扩展通过过度的网络权限截获了 90 万用户的 AI 对话。CtxPort 的架构设计从根本上杜绝了这种可能。

### Q4：商业模式是什么？

**Freemium + 开源核心。**

| 层级 | 价格 | 功能 |
|------|------|------|
| **Free** | $0 | 当前会话一键复制、左侧列表复制、基础 Markdown 格式、ChatGPT + Claude 支持 |
| **Pro** | $8/月 或 $68/年 | 批量多选复制、高级格式选项（仅用户/仅代码/精简版）、自定义 Bundle 模板、更多平台支持 |
| **Team** | $15/人/月 | Pro 全部功能 + 团队共享 Bundle 模板 + 管理后台 |

**定价逻辑**：

- Free 层足够解决单次复制的核心痛点，用于获客和口碑传播
- Pro 层解决批量操作和高级格式需求，这是高频用户的刚需
- $8/月 定价低于一杯咖啡，但产品每天为用户节省 30-60 分钟——ROI 极高
- 开源核心逻辑不影响商业化：免费用户只有基础功能，付费用户获得效率加速

**拉面盈利目标**：1000 个 Pro 用户 = $8,000 MRR = 拉面盈利。

### Q5：最大的风险是什么？

**按概率 x 影响排序，三个核心风险：**

**风险 1：平台 DOM 变更导致扩展失效（概率：高 | 影响：高）**
- ChatGPT 和 Claude 频繁更新前端代码，DOM 结构变化会直接导致内容提取失败
- **缓解措施**：Adapter 架构隔离变更影响范围；多层 CSS selector fallback；DOM 变更自动检测 + 热更新机制；社区贡献者帮助快速修复

**风险 2：平台封杀扩展（概率：中 | 影响：极高）**
- OpenAI 或 Anthropic 可能修改 ToS 明确禁止第三方扩展提取会话内容
- **缓解措施**：CtxPort 只处理用户自己的数据，用户有数据可移植性的法律权利（GDPR Article 20）；不触碰付费内容绕过、不突破 API rate limit；保持"用户数据权利工具"的叙事，而非"平台数据提取工具"

**风险 3：平台原生功能追赶（概率：中 | 影响：中）**
- ChatGPT 或 Claude 可能推出原生的结构化导出功能
- **缓解措施**：平台原生功能只会解决自家平台的导出，不会解决跨平台上下文迁移；CtxPort 的核心价值是"统一格式 + 跨平台"，这不是单个平台有动力做的；即使 ChatGPT 出了原生导出，用户仍需要 CtxPort 把它带到 Claude

### Q6：为什么现在是做这件事的最佳时机？

五个时机信号同时出现：

1. **"Context Engineering"成为行业共识**：Andrej Karpathy、Tobi Lutke 推动了从 prompt engineering 到 context engineering 的范式转移。Anthropic 发布了官方指南。市场教育成本接近于零。

2. **多平台并行使用成为常态**：60%+ 的 AI 用户同时使用多个平台（Triple-Stacking 行为）。上下文迁移不再是偶尔的需求，而是每天反复发生的核心工作流。

3. **安全事件创造信任真空**：2025 年 12 月的数据泄露事件（90 万用户受影响）让用户急需一个"可信赖的、本地处理的"替代方案。CtxPort 的零上传架构正好填补这个真空。

4. **Vibecoding 大众化扩大用户基数**：63% 的 vibecoding 用户是非开发者。这个群体对上下文管理的需求最大（因为他们缺乏手动管理的技术能力），却得到的工具支持最少。

5. **没有统治性竞品**：市场高度碎片化——每个平台一个导出工具、每种格式一个转换器。没有任何产品同时解决"跨平台、结构化、安全、批量"这四个维度。

## 3. Customer FAQ（用户 FAQ）

### Q1：我的数据会被上传吗？

**不会。绝对不会。**

CtxPort 的所有处理——HTML 解析、Markdown 转换、格式生成——都在你的浏览器本地完成。扩展不请求任何外部网络权限。你的会话内容永远不会离开你的电脑。

你可以自行验证：
1. 打开 Chrome DevTools → Network 面板
2. 使用 CtxPort 复制一个会话
3. 观察：零外部网络请求

此外，核心代码 100% 开源，任何人都可以审计。

### Q2：支持哪些平台？

**MVP 版本支持 ChatGPT 和 Claude 网页端。**

这两个平台覆盖了绝大多数 AI 用户。我们选择先把两个平台的体验做到极致，而不是广撒网但体验平庸。

后续计划支持：Gemini、DeepSeek、Grok。同时我们会开放 Adapter SDK，让社区贡献者添加更多平台支持。

### Q3：Context Bundle 是什么格式？

**人类可读的 Markdown。**

一个典型的 Context Bundle 长这样：

```markdown
<!-- CtxPort Context Bundle -->
<!-- Source: ChatGPT | Date: 2026-02-07 | Messages: 24 -->
<!-- URL: https://chat.openai.com/c/abc123 -->

# 讨论 REST API 认证方案

## User
我在做一个 SaaS 产品，需要选择 API 认证方案。候选是 JWT 和 OAuth2...

## Assistant
基于你的场景，我建议使用 OAuth2 + JWT 的组合方案...

## User
代码示例？

## Assistant
```python
# JWT Token 生成
import jwt
...
```
```

关键特性：
- **人类可读**：纯 Markdown，任何文本编辑器都能打开
- **机器友好**：结构化的元数据注释（来源、时间、消息数）、清晰的角色分段
- **代码保真**：代码块完整保留语言标记和缩进
- **可编辑**：复制到剪贴板后，你可以在任何编辑器中修改再使用

### Q4：免费还是付费？

**核心功能免费，高级功能付费。**

| 免费功能 | 付费功能（Pro $8/月） |
|---------|---------------------|
| 当前会话一键复制 | 批量多选复制 |
| 左侧列表复制按钮 | 高级格式（仅用户/仅代码/精简版） |
| 基础 Markdown 格式 | 自定义 Bundle 模板 |
| ChatGPT + Claude 支持 | 更多平台支持 |

大多数用户用免费版就够了。如果你每天频繁在 AI 工具间搬运上下文、需要批量操作，Pro 版会让你的效率提升一个数量级。

### Q5：如何安装使用？

**三步完成：**

1. **安装**：在 Chrome Web Store 搜索 "CtxPort"，点击"添加到 Chrome"
2. **使用**：打开 ChatGPT 或 Claude，你会看到：
   - 会话页面右上角出现 "Copy as Bundle" 按钮——点击即复制当前会话
   - 左侧会话列表每条旁边出现复制图标——点击即复制该会话（不用打开）
3. **粘贴**：在任何 AI 工具中 Ctrl+V / Cmd+V，粘贴结构化的 Context Bundle

不需要注册账号、不需要配置、不需要授权任何权限给第三方。打开就能用。

### Q6：和 ChatGPT 的原生"Export Data"功能有什么区别？

| 维度 | ChatGPT 原生导出 | CtxPort |
|------|-----------------|---------|
| 范围 | 全量导出所有历史 | 选择性复制特定会话 |
| 速度 | 异步邮件，数小时 | 即时复制，2 秒 |
| 格式 | JSON（需技术知识） | 干净的 Markdown |
| 操作 | 设置页面深入 3 层 | 页面内一键操作 |
| 用途 | 数据备份/合规 | 直接喂给其他 AI |

**ChatGPT 原生导出是为了合规备份设计的，CtxPort 是为了工作流效率设计的。** 两者解决不同的问题。

### Q7：扩展会不会影响 ChatGPT/Claude 的页面性能？

**几乎不会。**

CtxPort 只在你点击复制按钮时才激活内容提取逻辑。平时它只注入了几个轻量的 UI 按钮（复制图标），对页面性能的影响可以忽略不计。我们在内部测试中，包含 100+ 条消息的长会话的复制操作也能在 2 秒内完成。

## 4. MVP 边界

### 做什么（MVP 功能列表）

| # | 功能 | 说明 | 优先级 |
|---|------|------|--------|
| 1 | 当前会话一键复制 | 在 ChatGPT/Claude 会话页面点击按钮，复制为 Markdown Context Bundle | P0 |
| 2 | 左侧列表不打开就能复制 | 每条会话旁有复制图标，不用打开即可复制 | P0 |
| 3 | 批量多选复制 | 进入多选模式，勾选多条会话，一键合并为单一 Bundle | P0 |
| 4 | 复制格式选项 | 原文 / 仅用户消息 / 仅代码块 / 精简版 | P1 |
| 5 | 支持 ChatGPT 网页端 | 完整适配 chat.openai.com 的 DOM 结构 | P0 |
| 6 | 支持 Claude 网页端 | 完整适配 claude.ai 的 DOM 结构 | P0 |
| 7 | 本地处理，零上传 | 所有逻辑在浏览器本地执行 | P0 |
| 8 | 开源核心逻辑 | 内容提取和 Bundle 生成逻辑 MIT 开源 | P0 |
| 9 | 复制成功反馈 | 视觉 + 声音提示复制成功，包含消息数和估算 token 数 | P1 |

### 不做什么（明确排除项）

| # | 排除项 | 原因 |
|---|--------|------|
| 1 | Gemini / DeepSeek / Grok 支持 | 先深后广，两个平台足够验证核心假设 |
| 2 | GitHub repo 打包 | 不同产品形态（CLI），MVP 聚焦浏览器扩展 |
| 3 | Gmail 邮件复制 | 高隐私风险，且非 AI 上下文迁移的核心场景 |
| 4 | Twitter 内容复制 | 非核心场景，合规风险 |
| 5 | CLI 工具（ctxport CLI） | 第二阶段产品，需要独立的技术架构 |
| 6 | Token 预算/计数 | 有价值但非 Day 1 必须，延后到 v1.1 |
| 7 | 自动脱敏（邮箱/API Key 打码） | 有价值但增加复杂度，延后到 v1.1 |
| 8 | Web 应用/Dashboard | MVP 只做扩展，不做独立的 Web 界面 |
| 9 | 会话搜索/标签/分组 | 会话管理是独立产品方向，MVP 不做 |
| 10 | 导入功能 | MVP 只做"从 AI 平台复制出来"，不做"导入到 AI 平台" |

### 为什么这样划分

**一句话：砍到只剩核心假设验证所需的最小功能集。**

MVP 要验证的核心假设只有三个：

1. **用户是否真的需要结构化的 AI 会话复制？**（而非只需要纯文本复制）
2. **左侧列表不打开复制是否是杀手级功能？**（能否成为传播引爆点）
3. **零上传 + 开源是否能有效建立信任？**（在安全事件后用户是否愿意安装新扩展）

这三个假设只需要 ChatGPT + Claude 两个平台、一键复制 + 列表复制 + 批量复制三个核心功能就可以验证。更多的平台、CLI 工具、脱敏功能不改变这些假设的验证结果——它们是验证成功后的扩展方向。

**Bezos 原则：如果你不确定一个功能是否需要，答案就是不需要。**
